# ResearchLoop Kit

ResearchLoop Kit 是一套通用 AI 研究框架，用于长期、可沉淀、以证据为核心的研究任务。它不绑定某一个领域。用户只需要填写研究简报，AI 代理就会按固定研究循环推进：加载状态、审阅证据、提出假设、设计或执行实验、评估结果，并把经验沉淀到下一轮。

英文 README: [README.md](README.md)

## 为什么需要它

很多研究提示词会越写越臃肿，因为它们把领域事实、运行环境、实验流程、输出格式和记忆规则全塞进一个文件。这个项目把这些职责拆开：

- `AGENTS.md` 定义代理行为。
- `templates/RESEARCH_BRIEF.template.md` 承载领域输入。
- `templates/*` 保存可复用的产出格式。
- `state/` 保存长期研究记忆。
- `artifacts/` 保存报告、计划、结果日志和 provenance 记录。
- `skills/run-research-iteration/SKILL.md` 展示如何把研究循环封装成可复用 skill。

## 快速开始

1. 复制模板：

   ```bash
   cp templates/RESEARCH_BRIEF.template.md RESEARCH_BRIEF.md
   ```

2. 填写研究领域、问题、数据来源、成功指标、约束和交付物。

3. 在本目录运行代理：

   ```bash
   codex -p "$(cat AGENTS.md)"
   ```

4. 每次迭代把输出保存到 `state/` 和 `artifacts/`。

5. 发布前运行结构检查：

   ```bash
   bash scripts/verify.sh
   ```

## 核心循环

每次迭代包含十步：

1. 加载研究简报和历史状态。
2. 把目标重述为可衡量的成功标准。
3. 审阅当前证据并标注来源。
4. 提出一到三个可检验假设。
5. 选择一个小的实验、分析或综合任务。
6. 在用户约束范围内执行。
7. 按成功指标评估。
8. 沉淀决策、未解决问题和可复用经验。
9. 在结束迭代前核验引用的产物、检查结果和未支撑声明。
10. 生成下一轮具体计划。

## 研究完整性

- 最终或可复用产出应包含 provenance：接受或拒绝的来源、数据版本、命令、原始产物路径和核验状态。
- 除非研究简报明确允许，合成、仿真、模拟、随机或示例数据只能用于 smoke test，不能支撑研究结论。
- 优化类工作应先建立 baseline，并在固定预算或可比条件下比较结果。
- 大型或可恢复工作应把计划和原始笔记保存到 `artifacts/`，不要只依赖聊天上下文。

## 设计原则

- 基础提示词保持领域中立。
- 领域事实写进研究简报，不写进代理提示词。
- 使用渐进披露：`AGENTS.md` 保持短，详细格式放进模板。
- 用证据和验收门槛替代空泛激励。
- 区分“研究记忆”和“研究输出”。
- 每个结论都必须能追溯到数据、资料、实验输出，或明确标注为假设。

## 需要改哪里

大多数项目只需要编辑这些文件：

- `RESEARCH_BRIEF.md`：当前研究问题与约束。
- `state/research_log.md`：迭代历史。
- `state/leaderboard.md`：最佳结果或最强发现。
- `state/open_questions.md`：未解决问题。

不要为了每个领域频繁修改 `AGENTS.md`。如果提示词需要不断加入领域细节，应把这些内容移到研究简报或独立参考文件中。

## 发布说明

本项目刻意保持小而清晰，便于审计、复制和改造。参考资料中列出了影响本结构的公开材料，但本项目没有大段复制其他项目的提示词正文。
